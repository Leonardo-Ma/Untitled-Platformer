- Best practices:
	- Autoload (Singleton):
		- If it makes sense, it is possible to avoid creating a global singleton by using a **static instance** (Local singleton/self-registering singleton). It acts exactly as an Autoload (Globally accessible) but it is isolated to a scene. This makes it so when scene closes, the singleton is destroyed, freeing memory.
		- Use Autoloads for things that must survive between scene transitions: Sound Managers, Global Event Buses, Save Game Data, Goap AI action planner.
		- Use Static Instances for things that are vital globally, but only exist while a specific level/scene is loaded: UI Manager, Level Manager, specific PlayerController/boss controller of a boss fight.
