- Commit
	- Should follow [git style guide](https://github.com/agis/git-style-guide)
	- Should include
	- Preferably include basic unit test cases depending on mechanisms implemented
- Version
	- Starting from 1.0.0 Version naming follows: major.minor.patch (Ex 1.7.12)
	- Major includes breaking changes and big implementations, minor includes new implementations and possible breaking changes, patches are bugfixes and very small changes.
- Should pass github actions that check for styles and unit tests
- If performance is needed, code in C.
